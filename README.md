# Call Analyzer для СТО

Бот для аудиту дзвінків автосервісу: бере аудіозаписи з Google Drive,
транскрибує їх локально через faster-whisper, аналізує роботу менеджера через
локальну LLM (Ollama + Qwen 2.5) і формує звіт у Google Sheets з підсвічуванням
проблемних дзвінків.

**Усе працює локально** — нема API-ключів, нема платежів, дані клієнтів
не покидають вашу машину.

## Що робить пайплайн

1. Читає аудіофайли з вхідної папки на Google Drive.
2. Копіює їх у вашу робочу папку на Drive.
3. Транскрибує українською через faster-whisper.
4. Зберігає `.txt`-транскрипт поряд з аудіо у робочій папці.
5. Аналізує транскрипт через локальну LLM, отримує:
   - тип звернення, телефон, філію, імʼя менеджера
   - 8 бінарних оцінок по скрипту і збору інформації про авто
   - тип роботи з переліку "Топ-100 робіт СТО"
   - результат, оцінку 1-10, коментар
6. Дописує рядок у Google Sheets.
7. Якщо менеджер працював погано (мат, грубість, некоректна інформація) —
   рядок підсвічується червоним для подальшого реагування.

## Структура звіту

| Група | Колонки |
|---|---|
| Ідентифікація | Дата, Тип звернення, Номер телефону, Філія, Менеджер |
| Скрипт | Привітання/представлення (1/0) |
| Інфо про авто | Кузов, Рік, Пробіг (1/0 кожне) |
| ДОП продажі | Комплексна діагностика, Попередні роботи (1/0) |
| Скрипт | Запис на сервіс (дата), Прощання (1/0) |
| Інфо | Робота з топ-100, Дотримання інструкцій (1/0), Пропущені рекомендації |
| Результат | Результат, Оцінка 1-10, Запчастини, Коментар |
| **Сума балів** | автоматичний підрахунок 8 бінарних критеріїв |

Вісім бінарних колонок: представлення, кузов, рік, пробіг, комплексна
діагностика, попередні роботи, прощання, дотримання інструкцій топ-100.

## Перелік "Топ-100 робіт"

Жорстко закодований у `config.py` як константа `WORK_TYPES`. LLM зобовʼязана
обрати тільки зі списку — інакше нормалізатор поставить "інший варіант".

Зараз список: 22 пункти зі скрина шаблонної таблиці (Комп'ютерна діагностика,
Заміна оливи ДВЗ, Комплексна діагностика, Ендоскопія, ...). Якщо буде повний
список — замінити одну константу.

## Вимоги

- Python 3.10+
- ~16 GB RAM (для 7B-моделі), ~32 GB для 14B
- Сервісний акаунт Google Cloud з увімкненими Drive API та Sheets API
- [Ollama](https://ollama.com) для локальної LLM
- ~5-10 GB вільного місця на диску (faster-whisper модель + LLM)

## Інсталяція

### 1. Ollama + модель

**macOS / Linux:**
```bash
curl -fsSL https://ollama.com/install.sh | sh
```
**Windows:** інсталер з [ollama.com](https://ollama.com).

```bash
ollama serve              # сервер (часто стартує автоматично)
ollama pull qwen2.5:7b    # завантажує ~4.5 GB
ollama run qwen2.5:7b "Привіт"   # перевірка
```

> Альтернативи: `qwen2.5:14b` (краща якість), `llama3.1:8b` (трохи гірше з
> українською). Змінюється в `.env`.

### 2. Python-залежності

```bash
git clone <ваш-репозиторій>
cd call-analyzer

python -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate

pip install -r requirements.txt
```

### 3. Google Cloud

1. [Google Cloud Console](https://console.cloud.google.com/) → новий проект.
2. Увімкніть **Google Drive API** і **Google Sheets API**.
3. Створіть **сервісний акаунт** (IAM & Admin → Service Accounts).
4. Згенеруйте JSON-ключ → покладіть у `credentials/service-account.json`.
5. У Google Drive поділіться з email сервісного акаунту:
   - папкою з аудіо (доступ "Перегляд")
   - вашою робочою папкою (доступ "Редактор")

### 4. Конфіг

```bash
cp .env.example .env
```

Мінімум для запуску:
```env
GOOGLE_SERVICE_ACCOUNT_FILE=credentials/service-account.json
SOURCE_DRIVE_FOLDER_ID=1aB...    # ID папки з аудіо
WORK_DRIVE_FOLDER_ID=1cD...      # ID вашої робочої папки
OLLAMA_MODEL=qwen2.5:7b
```

`WORK_SHEET_ID` можна залишити порожнім — пайплайн створить нову таблицю
при першому запуску і виведе ID. Скопіюйте його в `.env`, щоб наступні
запуски писали в ту ж таблицю.

> **Важливо**: новостворена таблиця належить сервісному акаунту, а не вам.
> Щоб переглянути її — просто відкрийте URL з логів і поділіться з власною
> поштою через `Файл → Поділитися`. Або створіть таблицю руками і вкажіть її
> ID у `WORK_SHEET_ID`.

## Запуск

```bash
ollama list   # переконайтесь що qwen2.5:7b завантажений
python main.py
```

При першому запуску faster-whisper завантажить Whisper-модель (~3 GB для
`large-v3`). Якщо ноут слабкий — у `.env` поставте `WHISPER_MODEL=medium`
(~1.5 GB, швидше).

## Швидкість роботи (приблизно)

- **Транскрибація** (5-хв аудіо): ~30 сек на CPU, ~5 сек на GPU
- **LLM-аналіз** (1 транскрипт): ~10-20 сек на CPU, ~2 сек на GPU

20 файлів × 5 хв ≈ 15-20 хвилин на середньому ноуті без GPU.

## Як працює визначення проблемних дзвінків

LLM виставляє `is_problematic=true` коли:
- менеджер грубий, мат, нецензурна лексика
- дав некоректну/невірну інформацію
- не зміг допомогти і не запропонував альтернативу
- конфлікт або агресія з клієнтом
- очевидна непрофесійна поведінка

Такі рядки в Sheets фарбуються блідо-червоним (`rgb(1.0, 0.85, 0.85)`) для
зручного перегляду керівником.
